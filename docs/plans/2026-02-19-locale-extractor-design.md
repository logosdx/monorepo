# Locale Type Extractor CLI Design


## Overview

A CLI codegen tool inside `@logosdx/localize` that reads JSON locale files from a directory structure and generates a TypeScript interface + locale code union. The generated types plug directly into `LocaleManager<AppLocale, LocaleCodes>` for full `t()` autocomplete.

Approach: two new files in the existing package (`cli.ts`, `extractor.ts`), a `bin` entry in `package.json`, no external dependencies.


## Input Structure

Supports both flat and namespaced layouts in a single directory:

```
i18n/
├── en.json              # Root-level keys (no namespace prefix)
├── es.json
├── auth/
│   ├── en.json          # Keys nested under 'auth'
│   └── es.json
└── billing/
    ├── en.json          # Keys nested under 'billing'
    └── es.json
```

- Root `{code}.json` files → keys go to interface root
- `{namespace}/{code}.json` → keys nested under namespace key
- One locale (`--locale`, default `'en'`) is the "shape source" — other locales only contribute to the `LocaleCodes` union


## Output

Single `.ts` file:

```typescript
// Auto-generated by @logosdx/localize — do not edit

export interface AppLocale {
    errors: {
        required: string;
        invalid: string;
    };
    auth: {
        login: {
            title: string;
            subtitle: string;
        };
    };
    billing: {
        invoice: {
            total: string;
            due: string;
        };
    };
}

export type LocaleCodes = 'en' | 'es';
```


## CLI API

```bash
npx logosdx-locale extract \
    --dir ./i18n \
    --out ./src/locale-keys.ts \
    --locale en \
    --name AppLocale \
    --watch
```

| Flag | Required | Default | Purpose |
|------|----------|---------|---------|
| `--dir` | yes | — | Root directory to scan |
| `--out` | yes | — | Output `.ts` file path |
| `--locale` | no | `'en'` | Which locale's JSON to read for the type shape |
| `--name` | no | `'AppLocale'` | Name of the generated interface |
| `--watch` | no | `false` | Re-generate on file changes (`fs.watch`, 100ms debounce) |


## Core Algorithm

Three pure functions in `extractor.ts`:

### `scanDirectory(dir, locale)`

Walks the directory, returns structured data:

```typescript
interface ScanResult {
    rootShape: Record<string, unknown> | null;
    namespaces: Record<string, Record<string, unknown>>;
    codes: string[];
}
```

- Reads `dir/{locale}.json` for root shape
- Reads each subdirectory's `{locale}.json` for namespace shapes
- Collects all unique locale codes from all `.json` filenames found

### `jsonToInterface(shape, indent)`

Recursively converts a JSON object to TypeScript interface body:

- String/number values → `string` (translations are always rendered as strings)
- Nested objects → recurse
- Arrays → skip with warning

### `generateOutput(scanResult, interfaceName)`

Assembles the final `.ts` file:

- Header comment
- `export interface {name}` with root keys + namespace blocks
- `export type LocaleCodes = 'en' | 'es' | ...`

### Build

CLI entry point needs separate build step targeting Node (not browser). Emit `dist/cli.js` with `#!/usr/bin/env node` shebang.

### package.json

```json
{
    "bin": {
        "logosdx-locale": "./dist/cli.js"
    }
}
```


## Error Handling

| Scenario | Behavior |
|----------|----------|
| Missing `--dir` or `--out` | Print usage, exit 1 |
| `--locale` file not found in a namespace | Warn, skip that namespace |
| Malformed JSON | Error with file path, exit 1 |
| Empty directory | Empty interface, warn |
| Array value in JSON | Skip key with warning |


## Watch Mode

`--watch` flag enables `fs.watch` (recursive) on `--dir`. Changes are debounced (100ms) and trigger a full re-extract. Simple — no incremental diffing.


## Scope Boundaries

**Included:**
- JSON → TypeScript interface generation
- Flat + namespaced directory layouts
- Locale code union extraction
- Watch mode with debounce

**Not included:**
- Cross-locale validation (missing key detection is a linter concern)
- Deeply nested namespace directories (one level only)
- `.ts`/`.js` source files as input (JSON only)
- Runtime integration (purely codegen)
